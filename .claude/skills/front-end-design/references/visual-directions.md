# Visual Directions

Pick one direction and stay consistent. Do not mix directions on the same screen unless the user requests experimentation.

## 1. Editorial Grid

- Feel: structured, content-forward, typographic hierarchy.
- Type: high-contrast heading + neutral body text.
- Surfaces: clean cards, clear separators, minimal decoration.
- Motion: restrained, fade/slide transitions only.
- Best for: dense dashboards, skill lists, data review pages.

## 2. Utility Console

- Feel: operational, fast, status-focused.
- Type: compact UI scale with occasional monospace accents.
- Surfaces: bounded panels, clear rails, strong state colors.
- Motion: event-driven transitions (panel open, table updates).
- Best for: tool workflows, execution consoles, multi-step actions.

## 3. Premium Product

- Feel: expressive, polished, conversion-oriented.
- Type: stronger display typography and spacing rhythm.
- Surfaces: layered backgrounds, subtle gradients, intentional hero sections.
- Motion: staged reveal and section transitions.
- Best for: landing screens, onboarding, user-facing summaries.

## Selection Rule

- Use existing app language by default.
- Switch direction only if the user asks for a redesign or new product surface.
